```
Sample Scenario A: University Student Portal Integration
```
- The University of Progress is integrating a new Student Portal that connects:
- Existing Student Information System (SIS)
- Library Management System
- Learning Management System (Canvas)
- Payment Gateway for tuition fees

```
Interview Excerpts from Stakeholders (Problem Encountered):
```
- IT Director: "Our main concern is student data consistency. Currently, each system has different
    student ID formats."
- Registrar's Office: "We've heard some faculty are worried about grade submission through the
    new portal. They prefer the old method."
- Student Council President: "Students are excited but worried about technical issues during exam
    registration period."
- Library Staff: "We need proper training. The current system took 6 months for us to master."
- Finance Department: "Payment data must sync in real-time. Last semester, 15 students were
    wrongly marked as unpaid."

**Risk Identification**

- **Data Risk:** Student ID format inconsistency between systems
- **Data Risk:** Grade data syncing delays during peak submission periods
- **Process Risk:** Incomplete API documentation for integration
- **Process Risk:** No rollback plan if integration fails during semester start
- **People Risk:** Faculty resistance to new grade submission process
- **People Risk:** Library staff overwhelmed by complex new interface

**Justification:**

1. "Student ID mismatch has Impact=5 because wrong student data affects grades, payments, and records;
    Likelihood=4 because we have 4 different ID formats currently."
2. "Faculty resistance has Impact=4 because delayed grades affect transcripts; Likelihood=3 based on survey
    showing 40% faculty are hesitant."
3. "No rollback plan has Impact=5 (system failure during exams would be catastrophic) but Likelihood=2 (well-
    tested integration)."

**Mitigation Strategy Sample
Risk:** Faculty resistance to new grade submission process
**Actions:**

1. Identify 5 faculty champions for pilot program
2. Create simplified "cheat sheet" for common tasks
3. Set up faculty help desk with immediate support
    **Stakeholders:** Faculty champions, IT support, department heads
    **Timeline:** 2 weeks preparation, 4 weeks implementation, 2 weeks feedback collection
    **Success Metrics:**
- Quantitative: 80% faculty using system within 4 weeks
- Qualitative: Faculty satisfaction score ≥ 4/5 in survey
    **Barriers:** Senior faculty with technology anxiety
    **Solution:** One-on-one mentoring by faculty champions
